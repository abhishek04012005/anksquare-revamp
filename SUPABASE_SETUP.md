# Supabase Enquiry Table Setup

To store enquiries submitted through the popup form, create a table in your Supabase project. The following SQL will create the required structure:

```sql
-- run this in the SQL editor of your Supabase dashboard
create table public.enquiries (
  id bigint generated by default as identity primary key,
  name text not null,
  mobile text not null,
  service_type text not null,
  submitted_at timestamp with time zone default timezone('utc', now()) not null
);

-- optional: add an index on mobile or service_type if you need filtering
create index on public.enquiries (mobile);
```

## Environment Variables

Add the following values to your `.env.local` file at the root of the Next.js project:

```
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-public-key
```

You can find these in the "Settings → API" section of the Supabase dashboard.

## Supabase Client

We already created a client in `src/app/components/EnquiryModal.tsx` using `@supabase/supabase-js`. The component imports it directly:

```ts
const supabase = createClient(
  process.env.NEXT_PUBLIC_SUPABASE_URL as string,
  process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY as string
);
```

Submit an enquiry by invoking:

```ts
await supabase.from('enquiries').insert([{ name, mobile, service_type, submitted_at: new Date().toISOString() }]);
```

The modal handles success and error messages and resets the form.

## Notes

- Ensure you configure RLS and policies if you want to restrict inserts.
- Don't commit your `.env.local` to version control.
